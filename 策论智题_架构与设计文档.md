# 策论智题：自动化公考出题与智能训练系统

## 0. 项目命名

**推荐项目名：策论智题**

含义：
- **策**：政策文件、时政热点、基层治理、国家战略。
- **论**：申论、面试、政治理论表达能力。
- **智题**：基于 AI 自动采集、理解、生成、解析、评分、报告。

可选副标题：
> 面向公考、事业编、税务等考试的热点驱动型智能题库与训练系统

英文/技术代号：
> PolicyQuest AI / HotExam AI

备选名称：
1. **时政题库引擎**：偏工具化，适合内部系统。
2. **公考智练**：偏用户端产品，简单直接。
3. **热点智题**：突出“今年热点、最新文件”。
4. **策问公考**：适合做品牌，但技术感弱一点。
5. **青云题策**：有考试上岸寓意，适合 C 端。

最终建议：
> 对外产品名用 **策论智题**，内部系统代号用 **HotExam AI**。

# 一、项目架构文档

## 1. 项目背景

传统公考刷题产品的题库主要依赖人工编辑、历史真题和固定题库。其优势是稳定、规范，但缺点也明显：

1. 对**今年以来的新政策、新文件、新热点**响应较慢。
2. 言语理解、政治理论、面试题目与当年主流媒体表达、官方政策语境结合不够紧。
3. 面试训练往往只能给模板答案，缺少对用户个人表达的针对性评分、纠错和提升路径。
4. 用户刷题后通常只知道对错，缺少“为什么错、错在什么能力、下一步怎么补”的个人报告。

本项目希望建设一个“热点驱动型智能题库系统”：每天自动采集今年以来主流媒体、政府网站、权威政策文件等内容，经清洗、去重、聚类、摘要、知识化处理后，自动生成国考言语题、政治理论题、面试题及解析，并支持用户组卷训练、计时答题、自动评分、正确率统计和个人答题报告。

## 2. 项目目标

### 2.1 核心目标

建设一个具备以下能力的智能训练系统：

1. **自动采集**：按日采集主流媒体文章、政策文件、官方通报、地方治理案例。
2. **自动理解**：对文章进行清洗、去重、摘要、主题分类、热点聚类、关键词提取。
3. **自动出题**：根据文章内容生成国考言语题、政治理论题、面试题。
4. **自动组卷**：用户可以设置题型、数量、难度、热点范围、考试方向，系统自动组合练习。
5. **自动批改**：客观题自动判分，主观题/面试题按评分维度进行 AI 评分。
6. **自动解析**：每题提供出处、考点、解题思路、错因分析、延伸知识。
7. **能力画像**：统计用户正确率、题型薄弱点、热点掌握度、表达能力短板。
8. **个人报告**：生成日报、周报、阶段报告，给出针对性复习建议。
9. **计时训练**：支持单题计时、整套试卷计时、超时提醒。

### 2.2 产品差异化

对标粉笔、华图、中公等刷题产品，本项目的核心差异不是“历史题库更多”，而是：

> **用今年以来的新文章、新文件、新热点，自动生成更贴近当年考试语境的题目。**

差异化能力：

1. 热点更新更快。
2. 题目生成更灵活。
3. 面试评分更个性化。
4. 能围绕特定岗位方向定制训练，例如税务、基层治理、事业单位综合岗、行政执法岗。
5. 可以围绕山东、浙江、广东等地区政策素材生成地方化面试题。

1. 主流媒体公开文章。

4. 地方政府工作报告、新闻发布会材料。

### 合规建议

1. 优先使用 RSS、开放 API、授权数据源、公开转载许可内容。
2. 尊重 robots.txt 和网站服务条款。
3. 用户端展示时尽量展示摘要、出处链接、来源信息，不大规模复制原文。
4. 题目生成应保留来源追溯，便于审核和纠错。

### 题型范围

#### A. 国考言语题

1. 主旨概括题。
2. 意图判断题。
3. 细节理解题。
4. 标题填入题。
5. 语句排序题。
6. 语句衔接题。
7. 逻辑填空题。
8. 词句理解题。

#### B. 政治理论题

1. 单选题。
2. 多选题。
3. 判断题。
4. 填空题。
5. 简答题。
6. 政策理解题。
7. 时政热点题。
8. 文件精神理解题。

#### C. 面试题

1. 综合分析题。
2. 组织管理题。
3. 应急应变题。
4. 人际沟通题。
5. 现场模拟题。
6. 岗位认知题。
7. 政策落实题。
8. 税务专项题。
9. 基层治理题。
10. 青年干部成长题。

### 生成流程

```text
文章入库
  ▼
文章标签识别
  ▼
判断适合生成的题型
  ▼
调用题型 Prompt 模板
  ▼
生成题干、选项、答案、解析、考点、难度
  ▼
AI 自检
  ▼
规则校验
  ▼
重复检测
  ▼
进入待审核题库
  ▼
人工审核后正式入库
```

### 题目质检规则

1. 题干不能直接暴露答案。
2. 客观题必须有唯一正确答案。
3. 选项长度尽量均衡。
4. 干扰项必须有迷惑性，但不能明显错误或牵强。
5. 解析必须说明正确选项为什么对，错误选项为什么错。
6. 题目必须绑定文章来源、考点、题型、难度。
7. 面试题必须配套评分标准和示范作答。
8. 不生成涉密、侵权、低俗、偏激内容。

## 6.6 批改与评分模块

### 客观题评分

- 单选题：完全匹配得分。
- 多选题：可配置“少选给部分分、错选不得分”。
- 判断题：完全匹配得分。

### 面试题评分

建议采用“规则评分 + AI 评分 + 维度解释”的方式。

评分维度：

1. 审题准确度。
2. 观点明确度。
3. 逻辑结构。
4. 内容充实度。
5. 政策理解。
6. 案例结合。
7. 语言表达。
8. 岗位匹配度。
9. 可执行性。
10. 亮点与金句。

输出结果：

```text
总分：78/100
等级：中上
主要优点：观点明确，结构较完整，能提出措施。
主要问题：政策结合不足，案例不够具体，结尾升华偏模板化。
扣分点：
1. 审题没有回应“争议焦点”。
2. 措施偏泛，没有体现岗位职责。
3. 缺少本地案例或政策依据。
提升建议：
1. 开头先亮明态度。
2. 主体按“意义—问题—对策”展开。
3. 加入山东近两年政策案例。
示范答案：……
```

# 二、产品设计文档

## 1. 产品定位

**策论智题** 是一款面向公考、事业编、税务、选调生等考试的智能化刷题与面试训练产品。

它不是单纯的历史题库 App，而是一个能够持续追踪年度热点、政策文件、主流媒体文章，并自动转化为考试题目的 AI 训练平台。

核心价值：

> 让考生刷到更贴近今年热点、更贴近官方表达、更贴近真实考试方向的题目。

## 2. 目标用户

### 2.1 核心用户

1. 国考、省考考生。
2. 事业单位考生。
3. 税务系统考生。
4. 选调生考生。
5. 基层岗位、行政执法岗位备考者。

### 2.2 用户痛点

1. 不知道今年热点怎么考。
2. 刷题题库老旧，缺少新政策、新文件相关题目。

4. 做完题不知道自己到底弱在哪里。
5. 言语题会刷，但不会总结文章逻辑。
6. 政治理论记不住，也不知道怎么转化成题。
7. 面试答案空泛、模板化、缺少案例。
8. 复习没有数据反馈。

## 8. 风险与解决方案

## 8.1 题目质量风险

风险：AI 生成题目可能存在答案不唯一、解析牵强、题型不像真题。

解决：
1. 加入题目质检 Agent。
2. 建立规则校验。
3. 必须人工审核后上线。
4. 积累高质量题目模板。
5. 引入用户纠错反馈。

## 8.2 内容版权风险

风险：大规模采集并展示主流媒体文章可能产生版权问题。

解决：
1. 优先使用合法授权数据源。
2. 展示摘要而非全文。
3. 保留原文链接和来源。
4. 做内部训练素材与公开展示区分。
5. 必要时接入商业内容 API。

## 8.3 时政准确性风险

风险：AI 可能编造政策表述或错误理解文件精神。

解决：
1. 政治理论题必须绑定来源文件。
2. 文件类内容使用 RAG，不让模型凭空回答。
3. 解析中保留依据。
4. 敏感政策题进入人工审核。

## 8.4 成本风险

风险：文章多、题目多、用户多后，模型调用成本升高。

解决：
1. 分级模型策略。
2. 批处理。
3. 缓存。
4. 限制免费用户调用次数。
5. 只对高价值文章生成题目。

## 8.5 同质化竞争风险

风险：容易被认为只是另一个刷题系统。

解决：
1. 强化“今年热点自动出题”。
2. 强化“面试个性化评分”。
3. 强化“地方专项、岗位专项”。
4. 强化“热点专题 + 题目 + 报告”的闭环。

# 三、建议的 MVP 开发范围

如果你要尽快做出可用产品，不建议一开始就做得太大。第一版只做下面这个闭环：

```text
自动采集文章
  → 自动生成言语题/政治理论题
  → 后台人工审核
  → 用户设置题目数量开始刷题
  → 自动判分
  → 查看解析
  → 统计正确率和错题
```

第一版先不要做复杂 App、支付、机构端、语音面试。等题目质量和用户留存验证后，再加面试评分、个人报告和商业化。

第一版最关键的成败点只有三个：

1. **采集内容是否够新、够权威。**
2. **AI 生成题目是否像真题、答案是否严谨。**
3. **用户做完题后是否明显感觉“这比普通题库更贴近今年热点”。**
